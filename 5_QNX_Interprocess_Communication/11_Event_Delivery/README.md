# QNX Message Passing System — Event Delivery & `sigevent` Architecture

> **Companion to:** QNX Deadlock Avoidance (Part 2), Server Designs (Part 3), Multi-Part Messages (Part 4)

This document covers **event delivery** in QNX Neutrino — the mechanism by which threads, processes, and the kernel send asynchronous notifications to interested recipients. Events are the foundation of interrupt handling, timer expiry notifications, and inter-process asynchronous signaling.

---

## 1. What Are Events?

### Event as a Notification Container

An **event** in QNX is a structured notification that one entity (thread, process, or the kernel) wishes to deliver to another thread. Think of it as a **container** that holds the specification of *how* and *where* to deliver a notification when something interesting happens.

```
┌─────────────────────────────────────────────────────────┐
│                    Event Concept                        │
│                                                         │
│   ┌─────────────┐         ┌─────────────┐               │
│   │   Source    │         │  Recipient  │               │
│   │             │         │             │               │
│   │ Interrupt   │────────▶│  Thread A   │               │
│   │ Timer       │  Event  │  (waiting)  │               │
│   │ Server      │────────▶│             │               │
│   │ Kernel      │         └─────────────┘               │
│   └─────────────┘                                       │
│                                                         │
│   The event specifies:                                  │
│   • WHAT: Type of notification (pulse, signal, etc.)    │
│   • WHERE: Target channel, thread, or process           │
│   • HOW: Priority, value, code                          │
└─────────────────────────────────────────────────────────┘
```

### Sources of Events

| Source                 | Description                                             | Typical Use Case                                      |
| ---------------------- | ------------------------------------------------------- | ----------------------------------------------------- |
| **Hardware Interrupt** | Generated by physical hardware (NIC, UART, timer, GPIO) | Network packet arrival, serial data ready, timer tick |
| **Kernel Timer**       | Software timer managed by the kernel                    | Periodic tasks, timeouts, scheduling                  |
| **Thread-to-Thread**   | One thread notifies another within or across processes  | Server notifying client of async completion           |
| **Process-to-Process** | One process notifies another                            | Cross-process event signaling                         |

### Event vs Message vs Pulse

| Mechanism                  | Blocking?         | Size                      | Direction               | Use Case                                |
| -------------------------- | ----------------- | ------------------------- | ----------------------- | --------------------------------------- |
| **Message (`MsgSend`)**    | Yes (blocking)    | Variable (up to ~4GB)     | Client → Server         | Synchronous request/response            |
| **Pulse (`MsgSendPulse`)** | No (non-blocking) | 8 bytes (code + value)    | Any → Channel           | Async notification, hierarchy upward    |
| **Event (`sigevent`)**     | Configurable      | Structure (specification) | Any → Thread            | Configurable notification delivery      |
| **Signal**                 | Configurable      | Varies                    | Kernel/Process → Thread | Unix compatibility, simple notification |

> **Key Insight:** A `sigevent` is a **specification** of what notification to deliver. It can specify a pulse, a signal, or an interrupt unblocking. The event itself is not the notification — it describes *how* to generate the notification.

## 2. The `struct sigevent` Structure

### Structure Overview

The `struct sigevent` is the core data structure that defines an event notification. It is defined in `<sys/siginfo.h>` (and included via `<sys/neutrino.h>`).

```c
#include <sys/siginfo.h>
#include <sys/neutrino.h>

struct sigevent {
    int             sigev_notify;         		  // Notification type
    int             sigev_coid;           		  // Connection ID (for pulses)
    int             sigev_priority;     		  // Priority for delivery
    unsigned short  sigev_code;                   // Pulse code
    int             sigev_value;                  // Pulse value
    union {
        int         __sigev_tid;                  // Target thread ID
        void      (*__sigev_func)(union sigval);  // Thread callback
        struct {
            void  (*__sigev_func1)(union sigval);
            pthread_attr_t *__sigev_attr;
        } __sigev_thread;
        unsigned __sigev_event;         // Event number
    } __sigev_un;
    union sigval    sigev_value;        // Value to pass
    unsigned        sigev_flags;        // Flags (e.g., UPDATEABLE)
};
```

**Key fields:**

| Field            | Purpose                                                      |
| ---------------- | ------------------------------------------------------------ |
| `sigev_notify`   | Type: `SIGEV_PULSE`, `SIGEV_SIGNAL`, `SIGEV_INTR`, `SIGEV_THREAD` |
| `sigev_coid`     | Connection ID to target channel (for pulse delivery)         |
| `sigev_priority` | Priority at which the event is delivered                     |
| `sigev_code`     | Pulse code (application-defined, 8-bit)                      |
| `sigev_value`    | Pulse value (application-defined, 32-bit)                    |
| `sigev_flags`    | Modifiers: `SIGEV_FLAG_UPDATEABLE`                           |

### Initialization Macros

QNX provides convenience macros to properly initialize `sigevent` structures. **Always use these macros** rather than manual field assignment — they ensure correct initialization across QNX versions.

#### SIGEV_INTR_INIT

For **interrupt-type** events. The receiving thread unblocks via `InterruptWait()`.

```c
#include <sys/siginfo.h>

struct sigevent event;
SIGEV_INTR_INIT(&event);

// Usage: Register with kernel for interrupt notification
// Thread waits with: InterruptWait(0, NULL);
```

| Property       | Value                       |
| -------------- | --------------------------- |
| `sigev_notify` | `SIGEV_INTR`                |
| Unblock call   | `InterruptWait()`           |
| Source         | Kernel (hardware interrupt) |

#### SIGEV_PULSE_INIT

For **pulse-type** events. The receiving thread unblocks via `MsgReceive()` (with `rcvid == 0`).

```c
#include <sys/siginfo.h>

struct sigevent event;
SIGEV_PULSE_INIT(&event,
                 coid,           // Connection to target channel
                 priority,       // Delivery priority
                 code,           // Pulse code (8-bit)
                 value);         // Pulse value (32-bit)

// Usage: Deliver pulse to a channel
// Receiving thread: MsgReceive(chid, &pulse, sizeof(pulse), &info);
```

| Property       | Value                              |
| -------------- | ---------------------------------- |
| `sigev_notify` | `SIGEV_PULSE`                      |
| Unblock call   | `MsgReceive()`                     |
| Source         | Thread-to-thread, kernel-to-thread |

#### SIGEV_SIGNAL_INIT

For **signal-type** events. The receiving thread unblocks via signal delivery.

```c
#include <sys/siginfo.h>

struct sigevent event;
SIGEV_SIGNAL_INIT(&event, signo);  // e.g., SIGUSR1

// Usage: Deliver Unix signal to a thread
// Receiving thread: sigwait(), signal handler, or sigaction()
```

| Property       | Value                             |
| -------------- | --------------------------------- |
| `sigev_notify` | `SIGEV_SIGNAL`                    |
| Unblock call   | Signal handler, `sigwait()`, etc. |
| Source         | Kernel, thread-to-thread          |

#### SIGEV_THREAD_INIT

For **thread-callback** events. The kernel creates a new thread to run the specified function.

```c
#include <sys/siginfo.h>

struct sigevent event;
SIGEV_THREAD_INIT(&event,
                  func,       // void (*func)(union sigval)
                  value,      // Value passed to func
                  attr);      // pthread_attr_t (or NULL)

// Usage: Kernel creates thread running func(value) when event fires
```

| Property       | Value                            |
| -------------- | -------------------------------- |
| `sigev_notify` | `SIGEV_THREAD`                   |
| Unblock call   | New thread created automatically |
| Source         | Kernel (typically timers)        |

### Macro Summary Table

| Macro               | Event Type      | Unblock Call                 | Typical Source          |
| ------------------- | --------------- | ---------------------------- | ----------------------- |
| `SIGEV_INTR_INIT`   | Interrupt       | `InterruptWait()`            | Hardware interrupt      |
| `SIGEV_PULSE_INIT`  | Pulse           | `MsgReceive()`               | Thread-to-thread, timer |
| `SIGEV_SIGNAL_INIT` | Signal          | Signal handler / `sigwait()` | Kernel, compatibility   |
| `SIGEV_THREAD_INIT` | Thread callback | New thread spawned           | Timer expiry            |

> **Reference:** For the complete list of macros, see the QNX Library Reference under "s" → `sigevent`.

## 3. Event Types & Unblock Calls

### Interrupt Events

When hardware generates an interrupt, the kernel needs to notify a thread. The thread uses `InterruptWait()` to block until the interrupt occurs.

```
┌─────────────────────────────────────────────────────────┐
│              Interrupt Event Delivery                   │
│                                                         │
│  Hardware ──IRQ──▶ Kernel ──sigevent──▶ Thread          │
│                                                         │
│  Step 1: Thread initializes event                       │
│  struct sigevent event;                                 │
│  SIGEV_INTR_INIT(&event);                               │
│                                                         │
│  Step 2: Thread registers with kernel                   │
│  InterruptAttachEvent(irq, &event, flags);              │
│                                                         │
│  Step 3: Thread blocks waiting for interrupt            │
│  InterruptWait(0, NULL);  // BLOCKS HERE                │
│                                                         │
│  Step 4: Hardware generates IRQ                         │
│  Step 5: Kernel delivers event → thread unblocks        │
└─────────────────────────────────────────────────────────┘
```

```c
#include <stdio.h>
#include <stdlib.h>
#include <sys/neutrino.h>
#include <sys/siginfo.h>

// Interrupt service thread
void *interrupt_thread(void *arg) {
    int irq = *(int *)arg;  // e.g., IRQ for serial port
    struct sigevent event;
    int id;

    // Initialize event for interrupt notification
    SIGEV_INTR_INIT(&event);

    // Attach interrupt to this event
    id = InterruptAttachEvent(irq, &event, _NTO_INTR_FLAGS_TRK_MSK);
    if (id == -1) {
        perror("InterruptAttachEvent");
        return NULL;
    }

    printf("[IRQ Thread] Attached to IRQ %d\n", irq);

    for (;;) {
        // Block until interrupt occurs
        int ret = InterruptWait(0, NULL);
        if (ret == -1) {
            perror("InterruptWait");
            break;
        }

        printf("[IRQ Thread] Interrupt %d fired!\n", irq);

        // Handle interrupt (read hardware registers, etc.)
        // ...

        // Unmask interrupt to allow next one
        InterruptUnmask(irq, id);
    }

    InterruptDetach(id);
    return NULL;
}
```

### Pulse Events

Pulse events are the most common form of inter-thread notification in QNX. They are delivered to a channel and received via `MsgReceive()`.

```
┌─────────────────────────────────────────────────────────┐
│                Pulse Event Delivery                      │
│                                                         │
│  Sender Thread                    Receiver Thread       │
│  ┌─────────────┐                  ┌─────────────┐       │
│  │ Initialize  │                  │ Create      │       │
│  │ sigevent:   │                  │ Channel     │       │
│  │ SIGEV_      │                  │ chid =       │      │
│  │ PULSE_INIT  │                  │ ChannelCreate│      │
│  │ (coid=      │                  │ ()           │      │
│  │  receiver's │                  │              │      │
│  │  channel)   │                  │              │      │
│  └──────┬──────┘                  └──────┬──────┘       │
│         │                                │              │
│         │  MsgDeliverEvent() or          │  MsgReceive()│
│         │  MsgSendPulse()                │  (blocks)    │
│         └───────────────────────────────▶│              │
│                                          │              │
│                                          │  rcvid == 0  │
│                                          │  → Pulse!    │
│                                          └──────────────┘
└─────────────────────────────────────────────────────────┘
```

### Signal Events

Signal events provide Unix-compatible notification. They are less preferred in pure QNX designs but useful for porting or simple notifications.

```c
#include <signal.h>
#include <sys/siginfo.h>

struct sigevent event;
SIGEV_SIGNAL_INIT(&event, SIGUSR1);

// Thread waits for signal
sigset_t set;
sigemptyset(&set);
sigaddset(&set, SIGUSR1);
int sig;
sigwait(&set, &sig);
printf("Received signal %d\n", sig);
```

### Timer Events

Timers use `sigevent` to specify what happens when the timer expires.

```c
#include <time.h>
#include <sys/siginfo.h>

struct sigevent event;
struct itimerspec timer;
timer_t timerid;

// Initialize event: deliver pulse to our channel on expiry
SIGEV_PULSE_INIT(&event, coid, SIGEV_PULSE_PRIO_INHERIT,
                 MY_TIMER_CODE, 0);

// Create timer
timer_create(CLOCK_MONOTONIC, &event, &timerid);

// Set timer: 1 second initial, 500ms periodic
timer.it_value.tv_sec = 1;
timer.it_value.tv_nsec = 0;
timer.it_interval.tv_sec = 0;
timer.it_interval.tv_nsec = 500000000;
timer_settime(timerid, 0, &timer, NULL);

// Timer expiry will now deliver a pulse to our channel
```

## 4. Inter-Thread Event Delivery

### The Client-Server Event Pattern

This is the canonical pattern for asynchronous notification between processes in QNX:

```
┌─────────────────────────────────────────────────────────┐
│           Client-Server Event Registration Flow         │
│                                                         │
│  CLIENT                              SERVER             │
│  ┌─────────────┐                    ┌─────────────┐     │
│  │ 1. Create   │                    │ 1. Create   │     │
│  │    Channel  │                    │    Channel  │     │
│  │    (for     │                    │    (for     │     │
│  │    pulses)  │                    │    clients) │     │
│  └──────┬──────┘                    └──────┬──────┘     │
│         │                                   │           │
│  ┌──────▼──────┐                    ┌──────▼──────┐     │
│  │ 2. Connect  │────MsgSend────────▶│ 2. Receive  │     │
│  │    to       │    + sigevent      │    request  │     │
│  │    Server   │                    │             │     │
│  └─────────────┘                    └──────┬──────┘     │
│                                            │            │
│  ┌─────────────┐                    ┌──────▼──────┐     │
│  │ 3. Block on │◀───MsgReply────────│ 3. Verify   │     │
│  │    MsgRecv  │    "Registered"     │    event   │     │
│  │    (wait    │                    │    Store    │     │
│  │    for      │                    │    rcvid    │     │
│  │    pulse)   │                    │    Store    │     │
│  └─────────────┘                    │    event    │     │
│                                     └─────────────┘     │
│         ... time passes ...                             │
│                                                         │
│  ┌─────────────┐                    ┌─────────────┐     │
│  │ 4. Receive  │◀──MsgDeliverEvent──│ 4. Event    │     │
│  │    Pulse    │    (uses stored    │    occurs   │     │
│  │    rcvid=0  │     rcvid + event) │    Deliver  │     │
│  │    code=N   │                    │    pulse    │     │
│  └─────────────┘                    └─────────────┘     │
│                                                         │
│  5. Client handles pulse, may send new request          │
└─────────────────────────────────────────────────────────┘
```

### Step-by-Step Flow

**Step 1: Client initializes the event**

The client creates a channel to receive pulses and initializes a `sigevent` structure specifying pulse delivery to that channel.

```c
// Client code
int client_chid = ChannelCreate(0);  // Channel to receive pulses
int self_coid = ConnectAttach(0, getpid(), client_chid,
                            _NTO_SIDE_CHANNEL, 0);

struct sigevent event;
SIGEV_PULSE_INIT(&event,
                 self_coid,           // Deliver to our channel
                 SIGEV_PULSE_PRIO_INHERIT,  // Inherit priority
                 PULSE_CODE_NOTIFY,   // Application-defined code
                 0);                  // Initial value
```

**Step 2: Client registers the event with the server**

The client sends the event structure to the server as part of a registration message.

```c
struct reg_request {
    int msg_type;           // e.g., MSG_REGISTER_FOR_EVENTS
    struct sigevent event;  // The event to deliver
};

struct reg_request req;
req.msg_type = MSG_REGISTER_FOR_EVENTS;
req.event = event;  // Copy the initialized event

// Send to server
MsgSend(coid_to_server, &req, sizeof(req), &reply, sizeof(reply));
```

**Step 3: Server receives and verifies the event**

The server receives the registration, verifies the event is well-formed, and stores both the `rcvid` and the `sigevent`.

```c
struct reg_request req;
int rcvid = MsgReceive(chid, &req, sizeof(req), &info);

// Verify the event structure is valid
if (MsgVerifyEvent(rcvid, &req.event) == -1) {
    MsgError(rcvid, EINVAL);  // Bad event structure
    return;
}

// Store for later delivery
store_client_registration(rcvid, &req.event);

// Reply to unblock the client
MsgReply(rcvid, EOK, NULL, 0);
```

**Step 4: Server delivers the event when ready**

Later, when the event condition occurs, the server uses the stored `rcvid` and `sigevent` to deliver the notification.

```c
// When event occurs:
struct client_registration *reg = find_registration(client_id);
if (reg != NULL) {
    // Deliver the event (pulse) to the client's channel
    MsgDeliverEvent(reg->rcvid, &reg->event);
}
```

**Step 5: Client receives the pulse**

The client, blocked on `MsgReceive()`, receives the pulse.

```c
struct _pulse pulse;
struct _msg_info info;

for (;;) {
    int rcvid = MsgReceive(client_chid, &pulse, sizeof(pulse), &info);

    if (rcvid == 0) {
        // Pulse received!
        switch (pulse.code) {
            case PULSE_CODE_NOTIFY:
                printf("Event notification received! value=%d\n",
                       pulse.value);
                // Handle the event...
                break;
        }
    }
}
```

### Complete Code Example

```c
/*
 * event_delivery_demo.c
 * Demonstrates the complete client-server event registration
 * and delivery pattern using sigevent and MsgDeliverEvent.
 */

#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <pthread.h>
#include <sys/neutrino.h>
#include <sys/siginfo.h>

#define MSG_REGISTER_EVENT  1
#define MSG_GET_STATUS    2
#define PULSE_CODE_EVENT  10

// Registration request from client
struct reg_msg {
    int type;
    struct sigevent event;
};

// Stored client registration
struct client_reg {
    int valid;
    int rcvid;
    struct sigevent event;
    int client_pid;
};

#define MAX_CLIENTS 16
static struct client_reg clients[MAX_CLIENTS];
static pthread_mutex_t clients_mutex = PTHREAD_MUTEX_INITIALIZER;

/* ============================================================
 * SERVER
 * ============================================================ */

void store_registration(int rcvid, struct sigevent *event, int pid) {
    pthread_mutex_lock(&clients_mutex);
    for (int i = 0; i < MAX_CLIENTS; i++) {
        if (!clients[i].valid) {
            clients[i].valid = 1;
            clients[i].rcvid = rcvid;
            clients[i].event = *event;
            clients[i].client_pid = pid;
            printf("[Server] Stored registration for pid=%d at slot %d\n",
                   pid, i);
            break;
        }
    }
    pthread_mutex_unlock(&clients_mutex);
}

void deliver_events_to_all(void) {
    pthread_mutex_lock(&clients_mutex);
    for (int i = 0; i < MAX_CLIENTS; i++) {
        if (clients[i].valid) {
            printf("[Server] Delivering event to pid=%d\n",
                   clients[i].client_pid);

            // Update the value before delivery (if updateable)
            clients[i].event.sigev_value.sival_int = i + 100;

            int ret = MsgDeliverEvent(clients[i].rcvid, &clients[i].event);
            if (ret == -1) {
                perror("MsgDeliverEvent");
                // Client may have died — clean up
                clients[i].valid = 0;
            }
        }
    }
    pthread_mutex_unlock(&clients_mutex);
}

void *server_thread(void *arg) {
    (void)arg;
    int chid = ChannelCreate(0);
    int rcvid;
    struct reg_msg msg;
    struct _msg_info info;

    printf("[Server] Started, channel=%d\n", chid);
    printf("[Server] Connect with: ConnectAttach(0, %d, %d, ...)\n",
           getpid(), chid);

    for (;;) {
        rcvid = MsgReceive(chid, &msg, sizeof(msg), &info);
        if (rcvid == -1) {
            perror("MsgReceive");
            continue;
        }

        if (rcvid == 0) continue; // pulse

        switch (msg.type) {
            case MSG_REGISTER_EVENT: {
                // Verify the event is well-formed
                if (MsgVerifyEvent(rcvid, &msg.event) == -1) {
                    printf("[Server] Invalid event from pid=%d\n", info.pid);
                    MsgError(rcvid, EINVAL);
                    break;
                }

                printf("[Server] Valid event from pid=%d\n", info.pid);
                store_registration(rcvid, &msg.event, info.pid);
                MsgReply(rcvid, EOK, NULL, 0);
                break;
            }

            case MSG_GET_STATUS: {
                int count = 0;
                pthread_mutex_lock(&clients_mutex);
                for (int i = 0; i < MAX_CLIENTS; i++) {
                    if (clients[i].valid) count++;
                }
                pthread_mutex_unlock(&clients_mutex);
                MsgReply(rcvid, EOK, &count, sizeof(count));
                break;
            }

            default:
                MsgError(rcvid, ENOSYS);
                break;
        }
    }

    return NULL;
}

// Thread to periodically deliver events
void *event_generator_thread(void *arg) {
    (void)arg;
    for (int i = 0; i < 5; i++) {
        sleep(3);
        printf("[Server] Generating event batch %d...\n", i + 1);
        deliver_events_to_all();
    }
    return NULL;
}

/* ============================================================
 * CLIENT
 * ============================================================ */

void *client_pulse_receiver(void *arg) {
    (void)arg;
    int chid = ChannelCreate(0);
    struct _pulse pulse;
    struct _msg_info info;

    printf("[Client] Pulse receiver on channel=%d\n", chid);

    for (;;) {
        int rcvid = MsgReceive(chid, &pulse, sizeof(pulse), &info);
        if (rcvid == -1) {
            perror("MsgReceive");
            continue;
        }

        if (rcvid == 0) {
            printf("[Client] PULSE received! code=%d value=%d\n",
                   pulse.code, pulse.value);
        }
    }

    return NULL;
}

void run_client(int server_pid, int server_chid) {
    int coid;
    struct reg_msg req;
    struct sigevent event;
    int reply;
    int self_coid;

    // Create channel for receiving pulses
    int client_chid = ChannelCreate(0);
    self_coid = ConnectAttach(0, getpid(), client_chid,
                            _NTO_SIDE_CHANNEL, 0);

    // Start pulse receiver thread
    pthread_t receiver;
    pthread_create(&receiver, NULL, client_pulse_receiver, NULL);

    // Connect to server
    coid = ConnectAttach(0, server_pid, server_chid,
                         _NTO_SIDE_CHANNEL, 0);
    if (coid == -1) {
        perror("ConnectAttach");
        exit(EXIT_FAILURE);
    }

    // Initialize event for pulse delivery
    SIGEV_PULSE_INIT(&event,
                     self_coid,
                     SIGEV_PULSE_PRIO_INHERIT,
                     PULSE_CODE_EVENT,
                     0);

    // Make event updateable (server can modify value)
    SIGEV_MAKE_UPDATEABLE(&event);

    // Register event with server
    req.type = MSG_REGISTER_EVENT;
    req.event = event;

    printf("[Client] Registering event with server...\n");
    int status = MsgSend(coid, &req, sizeof(req), &reply, sizeof(reply));
    if (status == -1) {
        perror("MsgSend");
        exit(EXIT_FAILURE);
    }
    printf("[Client] Registration accepted!\n");

    // Wait for events
    pthread_join(receiver, NULL);
}

/* ============================================================
 * MAIN
 * ============================================================ */

int main(int argc, char *argv[]) {
    if (argc < 2) {
        printf("Usage:\n");
        printf("  %s server              (start server)\n", argv[0]);
        printf("  %s client <pid> <chid> (start client)\n", argv[0]);
        exit(EXIT_FAILURE);
    }

    if (strcmp(argv[1], "server") == 0) {
        pthread_t server, generator;
        pthread_create(&server, NULL, server_thread, NULL);
        sleep(1);  // Let server start
        pthread_create(&generator, NULL, event_generator_thread, NULL);
        pthread_join(server, NULL);
    } else if (strcmp(argv[1], "client") == 0) {
        if (argc < 4) {
            fprintf(stderr, "Usage: %s client <server_pid> <server_chid>\n",
                    argv[0]);
            exit(EXIT_FAILURE);
        }
        run_client(atoi(argv[2]), atoi(argv[3]));
    }

    return 0;
}
```

## 5. Event Registration & Security

### MsgRegisterEvent

For inter-process event delivery, the client must **register** its event with the kernel, binding it to a specific server connection. This prevents arbitrary processes from delivering events to the client.

```c
#include <sys/neutrino.h>

int MsgRegisterEvent(const struct sigevent *ev,
                     int coid);  // Connection to the server
```

**Purpose:**

- Tells the kernel: "Only the server at `coid` may deliver this event to me"
- Creates a security boundary for event delivery
- Required for thread-to-thread / process-to-process event delivery

**Not required for:**

- Kernel-to-thread events (interrupts, timers) — the kernel is implicitly trusted

```c
struct sigevent event;
SIGEV_PULSE_INIT(&event, self_coid, priority, code, value);

// Register with kernel: only server at 'coid' can deliver this event
int ret = MsgRegisterEvent(&event, coid_to_server);
if (ret == -1) {
    perror("MsgRegisterEvent");
}
```

### MsgUnregisterEvent

When the event is no longer needed, unregister it to free resources and remove the security binding.

```c
MsgUnregisterEvent(&event);
```

### Security: _NTO_REGEVENT_ALLOW_ANY_SERVER

By default, only the specific server (`coid`) can deliver the registered event. To allow **any** server to deliver the event (less secure):

```c
// Less secure: any server can deliver this event
int ret = MsgRegisterEvent(&event, _NTO_REGEVENT_ALLOW_ANY_SERVER);
```

| Flag                             | Security | Use Case                              |
| -------------------------------- | -------- | ------------------------------------- |
| `coid` (specific)                | High     | Normal operation — one trusted server |
| `_NTO_REGEVENT_ALLOW_ANY_SERVER` | Low      | Multiple servers may deliver events   |

> **Recommendation:** Always use a specific `coid` unless you have a compelling reason not to.

### SYSMGR_COID

To receive events from the **process manager** (a special system server), use `SYSMGR_COID`:

```c
// Register for events from the process manager
int ret = MsgRegisterEvent(&event, SYSMGR_COID);
```

This is used for system-level notifications such as:

- Process death notifications
- Resource manager events
- System configuration changes

## 6. Updateable Events

### SIGEV_FLAG_UPDATEABLE

By default, when a server delivers an event, it uses the **original** values stored in the `sigevent` structure at the time the client registered. However, sometimes the server needs to update values (e.g., the pulse value) at delivery time.

**Problem:**

```
Client registers:  sigevent with value = 0
Server stores:    the event (value = 0)
Time passes...
Server wants to deliver with value = 42 (updated)
Default behavior: value = 0 (original)
```

**Solution:** Client sets `SIGEV_FLAG_UPDATEABLE` to grant the server permission to modify the event at delivery time.

### SIGEV_MAKE_UPDATEABLE

Convenience macro to set the updateable flag:

```c
struct sigevent event;
SIGEV_PULSE_INIT(&event, coid, priority, code, initial_value);

// Grant server permission to update values
SIGEV_MAKE_UPDATEABLE(&event);

// Now server can modify event.sigev_value before delivery
```

### When & Why to Use

| Scenario                                              | Use Updateable? | Reason                              |
| ----------------------------------------------------- | --------------- | ----------------------------------- |
| Server needs to send different pulse values per event | Yes             | Each delivery can have unique value |
| Server only sends "something happened" (binary)       | No              | Original value is sufficient        |
| Server sends progress percentage                      | Yes             | Value changes each time             |
| Server sends error codes                              | Yes             | Different errors = different values |
| Simple heartbeat / keepalive                          | No              | Fixed value is fine                 |

**Server-side check:**

```c
// Server checks if it can update before delivery
if (event.sigev_flags & SIGEV_FLAG_UPDATEABLE) {
    // OK to modify
    event.sigev_value.sival_int = current_status;
}

MsgDeliverEvent(rcvid, &event);
```

## 7. Event Verification

### MsgVerifyEvent

Before storing a client's event, the server should verify that the event structure is well-formed and appropriate for the server's capabilities.

```c
#include <sys/neutrino.h>

int MsgVerifyEvent(int rcvid, const struct sigevent *ev);
```

**Return values:**

| Return | Meaning                          |
| ------ | -------------------------------- |
| `0`    | Event is valid and well-formed   |
| `-1`   | Event is invalid (check `errno`) |

**Why verify?**

```
┌─────────────────────────────────────────────────────────┐
│              Why MsgVerifyEvent Matters                  │
│                                                          │
│  Problem: Client sends malformed event                   │
│                                                          │
│  Client:                                                 │
│    struct sigevent ev;                                   │
│    memset(&ev, 0, sizeof(ev));  // Zeroed — BAD!       │
│    // Forgot to use SIGEV_PULSE_INIT                     │
│    ev.sigev_notify = 999;  // Invalid type               │
│                                                          │
│  Server (without verify):                                │
│    store_event(&ev);  // Stores garbage                  │
│    ...later...                                           │
│    MsgDeliverEvent(rcvid, &ev);  // KERNEL ERROR!        │
│    // System may crash or behave unpredictably           │
│                                                          │
│  Server (with verify):                                   │
│    if (MsgVerifyEvent(rcvid, &ev) == -1) {               │
│        MsgError(rcvid, EINVAL);  // Reject early         │
│        return;                                           │
│    }                                                     │
│    store_event(&ev);  // Guaranteed valid                 │
└─────────────────────────────────────────────────────────┘
```

**Verification checks:**

- Is `sigev_notify` a valid type?
- Is `sigev_coid` a valid connection?
- Is the target channel accessible?
- Are priority values in valid range?
- Is the event deliverable to the specified target?

### Why Verify?

| Without Verify                            | With Verify                                  |
| ----------------------------------------- | -------------------------------------------- |
| Server stores potentially invalid event   | Invalid events rejected at registration time |
| `MsgDeliverEvent()` may fail mysteriously | Client gets immediate error feedback         |
| Hard to debug delivery failures           | Clear error code (`EINVAL`)                  |
| Potential kernel instability              | Defensive programming                        |

## 8. Server Cleanup on Client Disconnect

When a client disconnects (crashes, exits, or calls `ConnectDetach()`), the server must clean up stored `rcvid`s and events. Otherwise, `MsgDeliverEvent()` will fail when attempted on a dead client.

### Detecting Disconnect

QNX provides a **disconnect pulse** (`_PULSE_CODE_DISCONNECT`) when a client detaches:

```c
#include <sys/neutrino.h>

for (;;) {
    int rcvid = MsgReceive(chid, &msg, sizeof(msg), &info);

    if (rcvid == 0) {
        struct _pulse *pulse = (struct _pulse *)&msg;

        if (pulse->code == _PULSE_CODE_DISCONNECT) {
            // A client has disconnected
            // Clean up any stored rcvids/events for this scoid
            int scoid = pulse->value;  // Server connection ID
            cleanup_client_by_scoid(scoid);
        }
        continue;
    }

    // Handle normal messages...
}
```

### Cleanup Strategy

```c
void cleanup_client_by_scoid(int scoid) {
    pthread_mutex_lock(&clients_mutex);
    for (int i = 0; i < MAX_CLIENTS; i++) {
        if (clients[i].valid && clients[i].rcvid_scoid == scoid) {
            printf("[Server] Cleaning up disconnected client slot %d\n", i);
            clients[i].valid = 0;
            // No need to unregister — kernel handles that
        }
    }
    pthread_mutex_unlock(&clients_mutex);
}
```

> **Note:** When using `name_attach()`, the name service automatically handles disconnect pulses. For raw channels, you must handle `_PULSE_CODE_DISCONNECT` manually.

## 9. Complete Working Example

### Full Event Delivery System

```c
/*
 * full_event_system.c
 * Complete demonstration of event delivery with:
 * - Multiple clients registering with a server
 * - Server verifying events
 * - Updateable pulse values
 * - Event delivery to all registered clients
 * - Cleanup on disconnect
 * 
 * Compile: qcc -o full_event_system full_event_system.c
 * Run:     ./full_event_system server    (Terminal 1)
 *          ./full_event_system client <pid> <chid>  (Terminal 2, 3, ...)
 */

#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <pthread.h>
#include <errno.h>
#include <sys/neutrino.h>
#include <sys/siginfo.h>

#define MSG_REGISTER    1
#define MSG_STATUS      2
#define PULSE_EVENT     42
#define MAX_CLIENTS     32

struct reg_request {
    int type;
    struct sigevent event;
};

struct client_entry {
    int valid;
    int rcvid;
    int scoid;
    int pid;
    struct sigevent event;
};

static struct client_entry g_clients[MAX_CLIENTS];
static pthread_mutex_t g_lock = PTHREAD_MUTEX_INITIALIZER;

/* ==================== SERVER ==================== */

void server_cleanup(int scoid) {
    pthread_mutex_lock(&g_lock);
    for (int i = 0; i < MAX_CLIENTS; i++) {
        if (g_clients[i].valid && g_clients[i].scoid == scoid) {
            printf("[Server] Client pid=%d disconnected, cleaning slot %d\n",
                   g_clients[i].pid, i);
            g_clients[i].valid = 0;
        }
    }
    pthread_mutex_unlock(&g_lock);
}

void server_register(int rcvid, struct sigevent *ev, struct _msg_info *info) {
    // Verify event before storing
    if (MsgVerifyEvent(rcvid, ev) == -1) {
        printf("[Server] Invalid event from pid=%d: %s\n",
               info->pid, strerror(errno));
        MsgError(rcvid, EINVAL);
        return;
    }

    pthread_mutex_lock(&g_lock);
    for (int i = 0; i < MAX_CLIENTS; i++) {
        if (!g_clients[i].valid) {
            g_clients[i].valid = 1;
            g_clients[i].rcvid = rcvid;
            g_clients[i].scoid = info->scoid;
            g_clients[i].pid = info->pid;
            g_clients[i].event = *ev;

            printf("[Server] Registered client pid=%d scoid=%d slot=%d ",
                   info->pid, info->scoid, i);

            if (ev->sigev_flags & SIGEV_FLAG_UPDATEABLE) {
                printf("(updateable)\n");
            } else {
                printf("(fixed)\n");
            }

            pthread_mutex_unlock(&g_lock);
            MsgReply(rcvid, EOK, NULL, 0);
            return;
        }
    }
    pthread_mutex_unlock(&g_lock);

    MsgError(rcvid, EAGAIN);  // No slots
}

void server_deliver_all(int value) {
    pthread_mutex_lock(&g_lock);
    for (int i = 0; i < MAX_CLIENTS; i++) {
        if (!g_clients[i].valid) continue;

        struct sigevent ev = g_clients[i].event;

        // Update value if client allows it
        if (ev.sigev_flags & SIGEV_FLAG_UPDATEABLE) {
            ev.sigev_value.sival_int = value;
        }

        int ret = MsgDeliverEvent(g_clients[i].rcvid, &ev);
        if (ret == -1) {
            printf("[Server] Delivery failed to pid=%d: %s\n",
                   g_clients[i].pid, strerror(errno));
            g_clients[i].valid = 0;  // Mark dead
        } else {
            printf("[Server] Delivered event (value=%d) to pid=%d\n",
                   value, g_clients[i].pid);
        }
    }
    pthread_mutex_unlock(&g_lock);
}

void *server_main(void *arg) {
    (void)arg;
    int chid = ChannelCreate(_NTO_CHF_DISCONNECT);
    int rcvid;
    union {
        struct reg_request reg;
        struct _pulse pulse;
    } msg;
    struct _msg_info info;

    printf("[Server] PID=%d CHID=%d Started\n", getpid(), chid);
    printf("[Server] Connect: ConnectAttach(0, %d, %d, ...)\n",
           getpid(), chid);

    for (;;) {
        rcvid = MsgReceive(chid, &msg, sizeof(msg), &info);
        if (rcvid == -1) {
            perror("MsgReceive");
            continue;
        }

        if (rcvid == 0) {
            // Pulse
            if (msg.pulse.code == _PULSE_CODE_DISCONNECT) {
                server_cleanup(msg.pulse.value);
            }
            continue;
        }

        switch (msg.reg.type) {
            case MSG_REGISTER:
                server_register(rcvid, &msg.reg.event, &info);
                break;

            case MSG_STATUS: {
                int count = 0;
                pthread_mutex_lock(&g_lock);
                for (int i = 0; i < MAX_CLIENTS; i++) {
                    if (g_clients[i].valid) count++;
                }
                pthread_mutex_unlock(&g_lock);
                MsgReply(rcvid, EOK, &count, sizeof(count));
                break;
            }

            default:
                MsgError(rcvid, ENOSYS);
                break;
        }
    }

    return NULL;
}

void *server_broadcaster(void *arg) {
    (void)arg;
    for (int round = 1; round <= 10; round++) {
        sleep(2);
        printf("[Server] === Broadcast round %d ===\n", round);
        server_deliver_all(round * 100);
    }
    return NULL;
}

/* ==================== CLIENT ==================== */

void *client_receiver(void *arg) {
    (void)arg;
    int chid = ChannelCreate(0);
    struct _pulse pulse;
    struct _msg_info info;

    printf("[Client] Pulse receiver on chid=%d\n", chid);

    for (;;) {
        int rcvid = MsgReceive(chid, &pulse, sizeof(pulse), &info);
        if (rcvid == -1) {
            perror("MsgReceive");
            continue;
        }

        if (rcvid == 0) {
            printf("[Client] EVENT: code=%d value=%d\n",
                   pulse.code, pulse.value);
        }
    }

    return NULL;
}

void run_client(int server_pid, int server_chid) {
    int coid;
    int client_chid;
    int self_coid;
    struct reg_request req;
    int status;

    // Create channel for pulses
    client_chid = ChannelCreate(0);
    self_coid = ConnectAttach(0, getpid(), client_chid,
                              _NTO_SIDE_CHANNEL, 0);

    // Start receiver thread
    pthread_t receiver;
    pthread_create(&receiver, NULL, client_receiver, NULL);

    // Connect to server
    coid = ConnectAttach(0, server_pid, server_chid,
                         _NTO_SIDE_CHANNEL, 0);
    if (coid == -1) {
        perror("ConnectAttach");
        exit(EXIT_FAILURE);
    }

    // Initialize updateable pulse event
    SIGEV_PULSE_INIT(&req.event, self_coid,
                     SIGEV_PULSE_PRIO_INHERIT,
                     PULSE_EVENT, 0);
    SIGEV_MAKE_UPDATEABLE(&req.event);

    // Register event
    req.type = MSG_REGISTER;
    printf("[Client] Registering event...\n");
    status = MsgSend(coid, &req, sizeof(req), NULL, 0);
    if (status == -1) {
        perror("MsgSend");
        exit(EXIT_FAILURE);
    }
    printf("[Client] Registered successfully!\n");

    // Keep running to receive events
    pthread_join(receiver, NULL);
}

/* ==================== MAIN ==================== */

int main(int argc, char *argv[]) {
    if (argc < 2) {
        printf("Usage:\n");
        printf("  %s server\n", argv[0]);
        printf("  %s client <pid> <chid>\n", argv[0]);
        exit(EXIT_FAILURE);
    }

    if (strcmp(argv[1], "server") == 0) {
        pthread_t srv, bcast;
        pthread_create(&srv, NULL, server_main, NULL);
        sleep(1);
        pthread_create(&bcast, NULL, server_broadcaster, NULL);
        pthread_join(srv, NULL);
    } else if (strcmp(argv[1], "client") == 0) {
        if (argc < 4) {
            fprintf(stderr, "Usage: %s client <pid> <chid>\n", argv[0]);
            exit(EXIT_FAILURE);
        }
        run_client(atoi(argv[2]), atoi(argv[3]));
    }

    return 0;
}
```

[Server] PID=1234 CHID=5 Started

## 10. Key Takeaways

### Event Concepts

| Concept       | Description                                                  |
| ------------- | ------------------------------------------------------------ |
| **Event**     | A specification (`sigevent`) of how/where to deliver a notification |
| **Source**    | Hardware interrupt, kernel timer, thread, or process         |
| **Delivery**  | Via `MsgDeliverEvent()`, kernel interrupt/timer infrastructure |
| **Reception** | Via `InterruptWait()`, `MsgReceive()`, signal handler, or new thread |

### sigevent Macros

| Macro               | Type      | Unblock Call      | Use                       |
| ------------------- | --------- | ----------------- | ------------------------- |
| `SIGEV_INTR_INIT`   | Interrupt | `InterruptWait()` | Hardware IRQ notification |
| `SIGEV_PULSE_INIT`  | Pulse     | `MsgReceive()`    | Thread-to-thread async    |
| `SIGEV_SIGNAL_INIT` | Signal    | Signal handler    | Unix compatibility        |
| `SIGEV_THREAD_INIT` | Thread    | New thread        | Timer callbacks           |

### Inter-Thread Event Flow

```
1. Client creates channel for pulses
2. Client initializes sigevent (SIGEV_PULSE_INIT)
3. Client sends event to server via MsgSend()
4. Server verifies event (MsgVerifyEvent)
5. Server stores rcvid + event
6. Server replies to unblock client
7. Later: Server delivers event (MsgDeliverEvent)
8. Client receives pulse via MsgReceive(rcvid==0)
```

### Security & Registration

| Call                             | Purpose                                          |
| -------------------------------- | ------------------------------------------------ |
| `MsgRegisterEvent()`             | Bind event to specific server (required for IPC) |
| `MsgUnregisterEvent()`           | Clean up registration                            |
| `_NTO_REGEVENT_ALLOW_ANY_SERVER` | Allow any server (less secure)                   |
| `SYSMGR_COID`                    | Receive events from process manager              |

### Updateable Events

| Flag                      | Behavior                                 |
| ------------------------- | ---------------------------------------- |
| None                      | Server uses original values at delivery  |
| `SIGEV_FLAG_UPDATEABLE`   | Server can modify values before delivery |
| `SIGEV_MAKE_UPDATEABLE()` | Convenience macro to set the flag        |

### Verification & Cleanup

| Call                     | Purpose                                       |
| ------------------------ | --------------------------------------------- |
| `MsgVerifyEvent()`       | Validate event structure at registration time |
| `_PULSE_CODE_DISCONNECT` | Detect client disconnect for cleanup          |
| `name_attach()`          | Automatically handles disconnect pulses       |

### Decision Matrix

| Scenario                | Event Type | Macro                              | Unblock Call      |
| ----------------------- | ---------- | ---------------------------------- | ----------------- |
| Hardware interrupt      | Interrupt  | `SIGEV_INTR_INIT`                  | `InterruptWait()` |
| Timer expiry → thread   | Pulse      | `SIGEV_PULSE_INIT`                 | `MsgReceive()`    |
| Timer expiry → callback | Thread     | `SIGEV_THREAD_INIT`                | New thread        |
| Server → Client async   | Pulse      | `SIGEV_PULSE_INIT`                 | `MsgReceive()`    |
| Unix signal needed      | Signal     | `SIGEV_SIGNAL_INIT`                | Signal handler    |
| Process manager event   | Pulse      | `SIGEV_PULSE_INIT` + `SYSMGR_COID` | `MsgReceive()`    |
